# 02-03 결재 대시보드 및 조회

## 1. 개요

결재 대시보드는 사용자가 자신과 관련된 결재 문서를 한눈에 파악하고 관리할 수 있는 화면이다. 기안함, 결재 대기함, 결재 완료함, 참조함을 제공하며, 문서 검색과 결재 통계 기능을 포함한다.

### 선행 의존성
- [02-01 결재 워크플로우](02-01-approval-workflow.md) - 결재 문서 데이터
- [02-02 문서 양식 관리](02-02-document-templates.md) - 양식별 필터링

## 2. 기능 요구사항

### 2.1 대시보드 요약
- [ ] 결재 대기 건수 (내가 결재해야 할 문서)
- [ ] 진행중 건수 (내가 기안한 문서 중 결재 진행중)
- [ ] 반려 건수 (최근 반려된 문서)
- [ ] 최근 완료 건수 (최근 7일 이내 완료)

### 2.2 문서함
- [ ] 기안함: 내가 기안한 문서 목록 (상태별 필터: 임시저장/상신/진행중/완료/반려)
- [ ] 결재 대기함: 내가 결재해야 할 문서 (현재 내 차례인 것만)
- [ ] 결재 완료함: 내가 결재 처리한 문서
- [ ] 참조함: 내가 참조로 지정된 문서 (읽음/안읽음 필터)

### 2.3 문서 검색
- [ ] 기간 검색 (상신일 기준)
- [ ] 제목 검색
- [ ] 기안자 검색 (이름/사번)
- [ ] 양식별 필터
- [ ] 문서 상태별 필터
- [ ] 긴급도별 필터

### 2.4 결재 통계
- [ ] 월별 기안 건수 / 처리 건수 (바 차트)
- [ ] 평균 처리 시간 (기안~최종 승인)
- [ ] 반려율
- [ ] 양식별 사용 빈도

## 3. 비기능 요구사항

- 대시보드 로딩 시간: 2초 이내
- 문서함 목록: 페이지네이션 (기본 20건)
- 실시간 알림 배지 업데이트 (WebSocket 또는 폴링)

## 4. 데이터베이스 스키마

이 태스크는 새로운 테이블을 생성하지 않으며, 02-01에서 정의한 테이블들을 조회한다.

### 4.1 주요 쿼리 인덱스 추가

| 테이블 | 인덱스 | 용도 |
|--------|--------|------|
| `approval_documents` | `idx_documents_submitted_by_status` (submitted_by, status) | 기안함 조회 |
| `approval_documents` | `idx_documents_status_submitted_at` (status, submitted_at) | 상태별 기간 조회 |
| `approval_lines` | `idx_lines_approver_status` (approver_id, status) | 결재 대기함 조회 |
| `approval_references` | `idx_references_user_read` (user_id, is_read) | 참조함 조회 |
| `approval_notifications` | `idx_notifications_user_read` (user_id, is_read) | 알림 조회 |

## 5. API 명세

### 5.1 대시보드

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/v1/approvals/dashboard/summary` | 대시보드 요약 (건수) |

**GET /api/v1/approvals/dashboard/summary**
```json
// Response (200)
{
  "pending_count": 3,
  "in_progress_count": 2,
  "rejected_count": 1,
  "recent_completed_count": 5
}
```

### 5.2 문서함

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/v1/approvals/drafted` | 기안함 |
| GET | `/api/v1/approvals/pending` | 결재 대기함 |
| GET | `/api/v1/approvals/completed` | 결재 완료함 |
| GET | `/api/v1/approvals/referenced` | 참조함 |

공통 쿼리 파라미터: `?page=1&size=20&status=&template_id=&date_from=&date_to=&keyword=`

**GET /api/v1/approvals/pending**

## 7. 인수 조건

- [ ] 대시보드에서 결재 대기/진행중/반려/완료 건수를 확인할 수 있다
- [ ] 기안함에서 내가 기안한 문서를 상태별로 필터링할 수 있다
- [ ] 결재 대기함에서 내 차례의 결재 문서만 표시된다
- [ ] 결재 완료함에서 내가 처리한 문서 이력을 볼 수 있다
- [ ] 참조함에서 읽음/안읽음 필터가 동작한다
- [ ] 기간, 제목, 기안자, 양식, 상태로 문서를 검색할 수 있다
- [ ] 월별 결재 통계 차트가 정상 렌더링된다
- [ ] 대시보드에서 문서를 클릭하면 결재 처리 페이지로 이동한다

## 8. 참고사항

- 대시보드 요약 데이터는 Redis 캐싱 고려 (30초 TTL)
- 문서함 목록은 서버사이드 페이지네이션으로 성능 확보
- 결재 대기 알림 배지는 헤더 네비게이션에 항상 표시
- 통계 차트는 shadcn/ui Charts 컴포넌트 사용 (Recharts 기반 래퍼)
